# M1 board handoff — uart_stream autonomous smoke

This is the handoff for Claude's board-side work. It does not claim a board
result.

## Build Inputs

- RM wrapper: `rtl/dfx/tpu_rp_rm_uart_stream.v`
- Island: `rtl/uart_stream_island_regs.v`
- Evaluator: `rtl/uart_stream_eval_core.v`
- Firmware entry: `sw/autoehw_board_mbox.c`
- Register map: `sw/uart_stream_regs.h`
- DFX scaffold: `scripts/vivado_dfx_uart_stream.tcl`

The RM keeps the zynq-ehw `tpu_rp` XBUS port contract. Inside NEORV32 firmware,
the island base is `0xF0000000`. Firmware publishes mailbox words to
`0xF1000000`; PS observes them through AXI GPIO at `0x41200000`.

## Host Gate

Run:

```sh
make all
```

Expected summary:

- 9 Python unittest cases pass.
- C twin, train-only runtime, firmware fake backend, and board mailbox host stub
  match the Python oracle.
- `uart_stream_eval_core` passes 144 Python-oracle RTL vectors.
- `uart_stream_island_regs` and `tpu_rp_rm_uart_stream` pass RTL smoke tests.

## Board Smoke Mailbox Golden

Board firmware constants:

- budget: `16`
- frames per condition: `8`
- search seed: `0xC0DE`

Expected mailbox sequence:

| Word | Meaning |
|---|---|
| `0xA7000000` | firmware reached `main` |
| `0xA8001008` | budget=16, frames=8 |
| `0xA90F05B7` | champion config: phase=15, majority=5, threshold=-73 |
| `0xAA013020` | train final: 19/32 |
| `0xAB011020` | final holdout: 17/32 |
| `0xAC000200` | done, evals=512 |

These are generated by the same Python oracle used by the host gate. The holdout
word is emitted only after the train-only search locks the champion config.

## Claude Steps

1. Run `make all`.
2. Run `make vivado-ooc`; update `docs/ooc_results.md` for
   `uart_stream_island_regs` and `tpu_rp_rm_uart_stream`.
3. Integrate `rtl/dfx/tpu_rp_rm_uart_stream.v` as a DFX RM in the proven zynq-ehw
   NEORV32 static shell, or complete `scripts/vivado_dfx_uart_stream.tcl`.
4. Build the NEORV32 firmware image from `sw/autoehw_board_mbox.c` with
   `autoehw_firmware.c`, `autoehw_runtime.c`, `autoehw_mmio_backend.c`, and
   `uart_stream_v1.c`.
5. Before `fpga loadb`, run the FCLK0=50 MHz preflight.
6. Load the bitstream, poll `0x41200000`, and record exact observations in
   `docs/board_results.md`.
